Programação#

O verdadeiro poder dos computadores, é claro, vem de programá-los. Saber programar, mesmo que um pouco, desbloqueia oportunidades inimagináveis de fazer o computador trabalhar melhor para você de maneiras mais pessoais. Além disso, habilidades de programação também desbloqueiam empregos muito bem remunerados. O advento da programação assistida por Machine Learning vai claramente impactar a arte de programar em geral, mas para ser assistido por ML/IA, você vai querer ter uma base sólida nos fundamentos. Neste capítulo, você experimentará um sistema de controle de versão chamado git, escreverá alguns programas simples em Python, encontrará recursos sobre aprendizado de máquina e será apresentado a um framework de aplicativos web chamado Django.

Acompanhando mudanças de qualquer coisa#

Mesmo que você nunca programe, você se beneficiará ao conhecer sistemas de controle de versão (VCS). Um VCS permite que você crie uma linha do tempo reversível de qualquer coisa em que você esteja trabalhando. Pense nisso como o recurso “Controlar Alterações” de um processador de texto, mas mais poderoso e para mais do que apenas documentos.

VCSs são geralmente associados a programadores porque eles realmente se beneficiam de ter uma linha do tempo reversível de seu software. Isso é útil muitas vezes, como quando um programa que costumava funcionar não funciona mais. Os programadores olham a linha do tempo das mudanças no código-fonte, retrocedem e reproduzem, enquanto executam seu programa para descobrir onde o erro foi introduzido.

Algumas pessoas mantêm seu currículo em um VCS. Elas querem ver seu progresso e também "ramificá-lo" para fazer versões ligeiramente diferentes para várias candidaturas de emprego.

Um VCS também é maravilhoso para colaboração. Você sabe como as pessoas enviam por e-mail versões de documentos como Proposta_FINAL_final_nt_jrc2_rascunho.docx? Um VCS pode tornar esse tipo de operação muito melhor. Todos podem trabalhar em sua própria cópia de um documento e então essas cópias podem ser mescladas. Um problema com essa abordagem é que o VCS funciona melhor com arquivos de texto puro, então se você quiser seguir por esse caminho, considere uma linguagem de marcação leve como ReST ou Markdown (veja Publicação).

Alguns VCSs notáveis incluem git, mercurial, subversion e Perforce. Git é um favorito bastante universal. É o epônimo do famoso GitHub, que a Microsoft adquiriu em 2018 por vários bilhões de dólares.

Usando git#

O Git tem uma leve reputação de ser difícil de aprender. Não tema, não é tão ruim. O Git foi feito para a linha de comando, mas pode ser usado graficamente se você preferir. Para os usos mais comuns, você só precisa saber alguns comandos. Aqui estão eles:

Alguns comandos chave do git#

Comando

Propósito

git init

Inicializar um novo repositório git na pasta atual

git clone

Fazer uma cópia local de um repositório remoto (por exemplo, do GitHub ou da sua equipe)

git add

Marcar um ou mais arquivos para fazer parte de um novo commit que está em processo

git commit

Tomar todas as mudanças marcadas e embalá-las como uma “mudança” discreta para ser lembrada para sempre.

git pull

Atualizar o repositório local com quaisquer mudanças que alguém tenha feito no servidor remoto.

git push

Enviar todos os nossos commits locais para o servidor remoto (para colaboração)

git log

Ver todas as mudanças que você ou qualquer outra pessoa fez

git checkout

Verificar um commit anterior ou uma ramificação separada (para voltar no tempo ou tentar caminhos alternativos)

A melhor maneira de aprender é experimentar e testar.

Instalando git#

SO

Comando

Em uma nova pasta vazia, execute git init para iniciar um novo repositório. Crie um novo arquivo de texto chamado file1.txt na pasta usando seu editor de texto e escreva algumas linhas de sua escolha. Por exemplo:

Tio Dave: Casado com Rose; gosta de minigolfe
Tia Malia: Foi para a Universidade de Michigan
Prima Jingjing: Do Havaí
Terceiro primo Pete: Gosta de cozinhar

Estagie-o com:

git add file1.txt

Faça o commit com:

git commit -m "Commit inicial de parentes"

Faça algumas alterações no arquivo. Exclua uma linha ou adicione uma nova. Veja as diferenças em relação ao último commit executando:

git diff

Estagie e faça o commit das novas alterações (repetindo os passos acima). Agora, digamos que você queira voltar à versão inicial do arquivo retrocedendo ao primeiro commit. Primeiro, você precisa descobrir qual é o “endereço” do commit, que é uma longa sequência de letras e números. Execute:

git log

para obter uma lista de commits. Copie as primeiras 8 ou mais letras/números do endereço do primeiro commit e preencha-os para sua versão do seguinte comando (seu endereço será diferente):

git checkout a3e2b54

Agora olhe para o arquivo e verá que ele está em seu commit original. Volte para o commit mais recente com:

git checkout master

Pronto, você usou a maioria das operações principais do git agora. Sem dúvida, alguns desses comandos ainda parecerão um pouco misteriosos. Há um livro inteiro gratuito sobre isso na página oficial do git chamado Pro Git e inúmeros tutoriais na internet. Faremos um exemplo semelhante, mas em um repositório colaborativo na próxima seção.

Além de trabalhar em arquivos de texto, há também uma extensão chamada Git Large File Storage (LFS) que pode ser usada para armazenar grandes ativos binários de forma eficiente em um repositório git (arquivos de design gráfico, arquivos de ciência, etc.). Git-annex, como aludido em Backups, pode rastrear informações sobre grandes conjuntos de arquivos sem armazenar os próprios arquivos no repositório git. Isso foi originalmente construído para gerenciar metadados e sincronização de bibliotecas de fotos e músicas. Está encontrando usos em áreas avançadas, como pesquisa de big data também.

Corrigindo algo em um repositório GitHub#

GitHub é um site (agora pertencente à Microsoft) que armazena milhares de repositórios git de projetos. Você pode clonar, puxar e enviar para colaborar na construção de software. Como exercício, vamos ver se podemos contribuir para um projeto de código aberto.

Primeiro, você precisará de uma conta no GitHub (use seu gerenciador de senhas para criar sua senha!). Em seguida, encontre um repositório para enviar uma solicitação de pull. Você pode criar seu próprio repositório e usá-lo, ou realmente fazer uma contribuição para a equipe de outro projeto. Uma maneira muito fácil de tentar seria procurar no GitHub por erros comuns de ortografia de uma palavra e, em seguida, fazer uma alteração para corrigi-la. É melhor encontrar erros de ortografia em comentários para não correr o risco de estragar o código. Melhor ainda, adicione alguma clarificação à documentação de um projeto que você tem usado um pouco.

Nota

Certifique-se de que o repositório tenha atividade recente, caso contrário, o mantenedor pode nunca responder à sua solicitação de pull.

Por que programar?#

Programar é o Santo Graal dos superpoderes digitais. Coisas como Machine Learning, jogos, processos de negócios, redes sociais, e assim por diante estão enraizadas na programação. Claro, é uma disciplina inteira que pode levar anos ou até décadas para realmente dominar, mas você pode fazer coisas significativas muito prontamente e seguir daí conforme sua situação. Se você quiser, pode fazer uma carreira bem remunerada inteira com isso. Ou você pode apenas se divertir.

Estou bastante convencido de que a programação deveria ser um tópico fundamental nas escolas de ensino médio até o ensino superior, possivelmente até substituindo algum currículo obrigatório de matemática que muito poucas pessoas acabam usando. Digo isso como um engenheiro de ciências exatas com doutorado: nunca tive que fazer divisão longa de polinômios e nunca terei. Mas, rapaz, já vi pessoas em todas as áreas da vida lutarem com coisas em que um pouco de exposição à programação teria ajudado.

A melhor maneira de aprender a programar é reconhecer que você tem um problema que deseja resolver e que um programa poderia ajudá-lo a resolvê-lo. Por exemplo, um amigo meu que agora é professor de matemática aplicada em Harvard me deu seu enigma matemático favorito uma vez:

Dadas duas 8’s e duas 3’s, combine-as com as quatro operações matemáticas básicas (+, −, ÷, ×) para obter 24. Você tem que usar cada uma. Por exemplo, 8+8+3+3 é um palpite válido, mas está errado porque isso equivale a 22, não 24. Da mesma forma, 8×3=24 é inválido porque você usou apenas um 8 e um 3.

Eu lutei por um tempo e decidi que um programa que adivinhasse todas as opções potenciais seria útil. No final, funcionou!

Mais praticamente, um amigo estava fazendo uma pesquisa médica usando um grande conjunto de dados de reabastecimento de farmácias e estava passando por esse processo tedioso e sujeito a erros em uma planilha para quantificar possíveis lacunas de medicação. Ela explicou seu processo em detalhes e então eu a ajudei a criar um pequeno programa que fazia o que ela estava fazendo. Assim, ela foi liberada para ajustar e depurar seu processo. Ela executou o programa dezenas de vezes, analisando o algoritmo que estava usando e chegando a conclusões interessantes.

Pessoalmente, passei muitos anos escrevendo programas que ajudam a simular reatores nucleares. Uma vez que temos um reator nuclear virtual, podemos ajustar seu design e realizar “experimentos” que custariam bilhões de dólares para executar no mundo real até acharmos que nosso design é ótimo. Temos que nos esforçar muito para garantir que os modelos correspondam à realidade (por exemplo, simulando coisas que realmente operaram e comparando os resultados com medições). A sinergia entre modelos físicos, simulações numéricas e experimentos é um verdadeiro cavalo de batalha na ciência e engenharia em geral.

Então, é claro, programadores profissionais constroem e mantêm todos os sistemas de software massivos por trás de uma grande fração de nossas experiências diárias.

Linguagens de programação#

Existem centenas de linguagens de programação. Cada uma delas tem o objetivo de traduzir (ou compilar) a entrada humana (código-fonte) em operações elétricas que o computador pode realizar. Suas diferenças surgem de diferentes compensações que seus criadores estavam visando:

  • É melhor ter um programa que roda muito rápido e pode falhar sem dizer por quê, ou um que roda devagar, mas diz em detalhes onde estava quando algo deu errado para ajudar na depuração?

  • O mesmo código-fonte deve rodar em muitos tipos de computadores, ou o programador deve manter diferentes versões para diferentes máquinas?

  • O programador deve esperar 10 minutos entre mudar o código-fonte e executar o programa para que o compilador possa otimizar agressivamente, ou deve ser instantâneo?

Linguagens de programação de nível muito baixo como assembly rodam extremamente rápido, mas requerem personalizações para cada modelo de CPU e são desafiadoras de escrever. Por outro lado, linguagens de alto nível como JavaScript são agradáveis de ler, mas geralmente lentas. Uma das linguagens mais recentes e muito populares, Rust, tenta permitir que programadores escrevam código muito “seguro” (se compilar, terá menos bugs do que o normal) enquanto ainda roda extremamente rápido e em paralelo.

Entre as indústrias de ciência de dados, automação, web e engenharia (e muitas outras!), a linguagem Python tem se saído muito bem na última década ou duas. É uma linguagem de alto nível interpretada que roda devagar por si só. Mas é tão popular que as pessoas a interligaram bem com algumas bibliotecas matemáticas, gráficas e de aprendizado de máquina de nível inferior e extremamente rápidas, tornando-a um bom cavalo de batalha. Há também uma vibrante coleção de outros módulos que as pessoas criaram que fazem todo tipo de coisa como análise estatística, criação de gráficos e aplicativos web. Aqui, vamos experimentar o Python. Dependendo do seu objetivo final, outra linguagem pode ser melhor.

Um programa para aproximar \(\pi\)#

Vamos fazer um programa real que faz algo interessante: aproxima \(\pi\) com o método de Monte Carlo. Isso aproveita o fato de que computadores não podem se entediar para aproximar uma constante fundamental da natureza basicamente jogando dardos.

Imagine que você está vendado e jogando dardos em um tabuleiro perfeitamente quadrado com comprimento de lado de 1 m e com um círculo desenhado nele que toca apenas as bordas. Se você jogar aleatoriamente, o número de dardos que caem dentro do círculo será proporcional à área do círculo, enquanto o número de dardos que caem em qualquer lugar do tabuleiro será proporcional à área do quadrado. Se você jogar milhões e milhões de dardos, essas aproximações se tornarão muito boas, e suas contagens dirão o que é \(\pi\).

O tabuleiro de dardos

\[\begin{split}\text{Área do Quadrado} &= s^2 \\ \text{Área do Círculo} &= \pi \left(\frac{s}{2}\right)^2\end{split}\]

Dividindo essas equações, obtemos:

\[ \begin{align}\begin{aligned}\begin{split}\frac{\text{Área do Quadrado}}{\text{Área do Círculo}} = \frac{4}{\pi} \\\end{split}\\\begin{split}\text{ou} \\\end{split}\\\pi = 4 \times \frac{\text{Área do Círculo}}{\text{Área do Quadrado}}\end{aligned}\end{align} \]

Em vez de jogarmos os dardos nós mesmos, podemos escrever um programa para simular os lançamentos de dardos e aproximar a razão das áreas. Para fazer isso, colocamos o seguinte em um novo arquivo de texto chamado darts.py. Observe que todas as linhas que começam com # são apenas comentários e não são executadas (então você não precisa digitá-las).

import random  

RADIUS = 1.0   # define uma constante

num_in_circle = 0 # inicializa nossa contagem

# pergunta ao usuário quantos dardos lançar
num_darts = int(input('Número de dardos: '))

for _individual_dart in range(num_darts):
    # para cada lançamento de dardo, pegue dois números aleatórios
    # entre 0 e 1.
    x = random.random()
    y = random.random()

    # Alguém se lembra da fórmula da distância? 
    # Podemos pular sqrt aqui porque sqrt(1) = 1
    d = x**2 + y**2
    if d < RADIUS:
        # incrementa a contagem no círculo
        num_in_circle = num_in_circle + 1

pi = 4*num_in_circle/float(num_darts)
print('Pi é aproximadamente: {}'.format(pi))

Execute o programa com $ python darts.py no prompt de comando normal, não no do Python. Se ainda vir >>> na sua linha de comando, execute exit(). Ele deve dar:

$ python darts.py
Digite o número de dardos: 100000
Pi é aproximadamente: 3.14504

Bem próximo! A resposta correta é 3.1415926.... Se você lançar apenas alguns dardos, a resposta será muito mais errada (tente!). Ah, o teorema central do limite em ação!

Meu computador leva apenas um segundo para fazer 1 milhão de lançamentos de dardos, mas não faça muitos, pode demorar muito! Lembre-se: Ctrl-C irá abortar um programa em execução se você ficar preso.

O programa só amostra números entre 0 e 1, então estamos realmente operando apenas no primeiro quadrante do tabuleiro de dardos (superior esquerdo). Isso é OK porque o tabuleiro é simétrico.

O que fazer se quisermos plotar nossos resultados? Precisaremos de algumas bibliotecas de terceiros para fazer isso (coisas que fazem parte da comunidade Python, mas não estão incluídas no próprio Python). Para obtê-las, use o gerenciador de pacotes Python, que é chamado pip. É muito semelhante aos gerenciadores de pacotes do sistema que temos usado, mas apenas puxa módulos Python. Precisamos de duas coisas para ter plotagem:

pip install --user numpy
pip install --user matplotlib

Precisamos modificar o programa para acompanhar as coordenadas (x, y) e, em seguida, plotá-las no final. Então, faremos listas para armazenar cada coordenada. Aqui está (desta vez como darts_plot.py):


import random  

import matplotlib.pyplot as plt

RADIUS = 1.0   

coords_inside = []
coords_outside = []

num_darts = int(input('Número de dardos: '))

if num_darts > 1000000:
    # verificação de erro
    raise RuntimeError('Muitas partículas.')

for _individual_dart in range(num_darts):
    x = random.random()
    y = random.random()

    d = x**2 + y**2
    if d < RADIUS:
        # adiciona à lista de coords dentro do círculo
        coords_inside.append((x,y)) 
    else:
        # também rastrear forasteiros para plotagem
        coords_outside.append((x,y)) 
pi = 4*len(coords_inside)/float(num_darts)
print('Pi é aproximadamente: {}'.format(pi))

# Fazer plotagem
# descompactar x's e y's em suas próprias listas
xinside, yinside = zip(*coords_inside) 
plt.plot(xinside, yinside, 'b.',label='Dentro')

xoutside, youtside = zip(*coords_outside) 
plt.plot(xoutside, youtside, 'gx',label='Fora')
plt.grid()
plt.legend()
plt.title('Aproximação de $\pi$')
plt.xlabel('x (m)')
plt.xlabel('y (m)')
plt.show()
# descomentar para salvar como arquivo
#plt.savefig('darts.png', dpi=200)

Uma regressão linear#

Python é ótimo para estatísticas. Isso é frequentemente útil para suas equipes de pesquisa e em casa (uma vez usei regressões para estimar a que horas o aquecedor precisava se ligar para atingir a temperatura desejada, dada a temperatura externa a cada manhã durante o inverno em Michigan). Vamos fazer um exemplo simples de leitura de dados diretamente do Excel. A biblioteca pandas fornece capacidades de análise de dados de alto nível, mas vamos usá-la apenas para ler do Excel:

pip install --user pandas
pip install --user xlrd

Precisamos de alguns dados para processar. Abra seu programa de planilhas favorito (Excel ou Open Office Calc funcionam bem, por exemplo) e vamos gerar alguns dados ruidosos. Vamos usar uma linha desta fórmula:

\[\begin{split}y = mx + b + \epsilon\\\end{split}\]

onde:

\[\begin{split}m &= 3.5 \\ b &= 2.3 \\ \epsilon &= \text{Ruído uniforme de -4 a 4}\end{split}\]

Nota

Se você tiver seus próprios dados nos quais deseja fazer uma regressão, basta carregá-los em vez disso!

Em uma nova planilha, rotule as duas primeiras colunas como X e Y. Na primeira coluna, insira a fórmula: =RAND()*5 para obter valores aleatórios de x entre 0 e 5. Clique na pequena caixa preta no canto inferior direito da célula e arraste para baixo 30 ou mais células para obter muitos valores aleatórios de X. Em seguida, na coluna Y, insira nossa linha como =3.5*A1 + 2.3 + (RAND()*8-4) [3]. Clique duas vezes na pequena caixa preta para preencher para baixo. Salve o arquivo como data.xlsx. Ele ficará assim:

Para fazer a regressão:

from scipy import stats
import pandas

# Ler dados
data = pandas.read_excel('data.xlsx')

# Pegar as colunas que queremos
x = data['X'].values
y = data['Y'].values

# Calcular a regressão
results = stats.linregress(x,y)

# descompactar resultados em nomes úteis
(slope, intercept, r_val, p_val, std_err) = results

# Exibir resultados
print('Inclinação: {}'.format(slope))
print('Intercepto: {}'.format(intercept))
print('R-quadrado: {}'.format(r_val**2))
print('P-val:     {}'.format(p_val))

Aqui está o resultado com meus dados:

Inclinação:     3.473511906229649
Intercepto: 1.8192562724560553
R-quadrado: 0.861400909580644
P-val:     1.1563714298035524e-12

É tudo o que precisa para fazer uma regressão! Podemos plotá-la também, adicionando este código:

# Fazer um gráfico enquanto estamos nisso. 
import matplotlib.pyplot as plt
import numpy
modelX = numpy.linspace(0,5,10)
modelY = modelX * slope + intercept
fig = plt.figure(dpi=300)
plt.plot(x,y,'o', label='Dados')
plt.plot(modelX, modelY, '--',label='Modelo')

# Adicionar rótulos e formatação
plt.title('Regressão bonita')
plt.xlabel('X')
plt.ylabel('Y')
plt.grid(linestyle='--')
plt.legend()
plt.text(3.1,3.0,'m={:.3f}'.format(slope))
plt.text(3.1,2.0,'b={:.3f}'.format(intercept))

plt.savefig('regression.png')

Ei, nada mal! Note que, ao definir o gráfico em código, podemos nos dar ao luxo de gastar tempo polindo sua aparência para ser muito profissional, porque só precisamos fazer essa parte uma vez. Podemos então lançar um bilhão de conjuntos de dados diferentes neste programa e não precisar gastar um segundo plotando os resultados; já fizemos isso uma vez, o computador cuida disso daqui em diante.

A ponte para o aprendizado de máquina#

De carros autônomos a reconhecimento facial ao Exterminador do Futuro, o aprendizado de máquina (ML) é um tópico bastante quente. A razão pela qual se tornou tão bom é que as placas gráficas de computadores se tornaram extremamente rápidas em fazer multiplicação matriz-vetor, impulsionadas principalmente pela indústria de jogos (onde essa matemática é necessária para sombreamento à medida que os jogadores se movem). Combine isso com a função universal (capaz de mapear qualquer entrada arbitrariamente complexa para uma saída arbitrariamente complexa) que é a Rede Neural Convolucional e conjuntos de dados massivos para treiná-las (por exemplo, vastas quantidades de selfies) e você tem uma revolução do aprendizado de máquina.

Uma boa maneira de colocar as mãos na massa em ML é se dedicar a alguns cursos fast.ai. Esses cursos online gratuitos são de ponta e exigem cerca de 8 horas por semana durante dois meses ou mais. O computador de todo mundo não vai ser bom para treinar em grandes conjuntos de dados (a menos que você seja um jogador), mas você pode tentar (eu consegui meu computador funcionando com eles). Melhor ainda, eles configuraram um ambiente na nuvem na Amazon Web Services onde você pode simplesmente fazer login em computadores "na nuvem" que são mais do que capazes de fazer os exercícios da aula.

Nota

Grande parte do trabalho de ML é feito através do Python, que interage com bibliotecas de peso por baixo dos panos. Essa é outra boa razão para escolher Python como primeira linguagem de programação.

Uma introdução mais simples à visão computacional (relacionada ao ML) é brincar com a biblioteca OpenCV. Ela tem boas ligações com Python e você pode até mesmo fazê-la funcionar em um mini-computador Raspberry Pi de $35. Já vi pessoas fazerem leitores de placas de carros, reconhecerem rostos de familiares, analisarem vídeos de experimentos para coletar dados e muitas outras coisas úteis.

Um exemplo trivial (não ML) de uso é obter alguns dados de um arquivo de filme. Eu tinha um vídeo de alta velocidade de um moinho de luz e queria ver quão rápido ele estava girando analisando o vídeo [2]. OpenCV é perfeito para isso. Descobri qual pixel eu queria observar e então extraí a intensidade versus tempo com um programa simples:

O código (parcial) para ler isso é assim:

import cv2  # trazer a biblioteca OpenCV

def measureIntensities(videoFileName, pointOfInterest):
    """Encontrar a intensidade de um ponto de interesse em cada quadro"""
    video = cv2.VideoCapture(videoFileName)
    intensities = []
    while video.isOpened():
        _returnCode, frame = video.read()
        if frame is None:
            break
        pixelValues = frame[pointOfInterest]
        intensities.append(pixelValues.sum())
    video.release()
return intensities

Se você tiver vídeo de algo e quiser quantificar algum elemento dele com um computador, OpenCV é seu ingresso. Uma vez que você esteja confortável com coisas simples como esta, pode tentar reconhecimento facial e leitura de placas de carros com ele também.

Interfaces Gráficas de Usuário#

Fazer aplicativos de desktop com GUI com diálogos, botões, menus e formulários pode tornar seus programas mais acessíveis e amigáveis. A maioria das linguagens de programação possui frameworks de GUI que fazem a maior parte do trabalho árduo de criar tais interfaces. Em Python, temos Kivy, wxPython, PyQt, Tkinter, e dezenas de outros. Eu escolhi wxPython primeiro por causa de seu excelente aplicativo de demonstração que permite experimentar todos os seus centenas de widgets interativamente com código de exemplo. Kivy é legal porque pode até rodar em celulares. PyQt é especialmente polido, embora exija uma licença comercial se usado em software comercial (não que haja algo de errado com isso).

Talvez o sistema de GUI mais universal e moderno seja um navegador da web, o que nos leva ao próximo tópico.

Aplicações Web#

Quase todas as páginas da web que você visita hoje em dia são na verdade uma aplicação web, o que significa que está executando algum programa nos bastidores e apresentando os resultados a você por meio de um navegador como Firefox ou Chrome. Quase sempre, há um banco de dados no back-end contendo as informações. Por exemplo, o Facebook é conceitualmente uma visão baseada na web dos dados do usuário armazenados em um banco de dados e regidos por regras e processos.

As possibilidades com aplicativos web são quase infinitas. Negócios e serviços inimagináveis ainda a serem criados serão aplicativos web. Um inventário doméstico poderia ser um aplicativo web. O Twitter é um aplicativo web. O Reddit é um aplicativo web. Os programas que dirigem aplicativos web são escritos em muitas variedades de linguagens de programação, assim como aplicativos offline. Ao longo dos anos, no entanto, vários frameworks evoluíram que contêm peças comumente necessárias de aplicativos web, como autenticação de usuários, envio de formulários seguro e acesso seguro a bancos de dados. Começar com um framework web é uma ótima maneira de se tornar produtivo na criação de aplicativos web seguros e performáticos rapidamente. Os centenas de frameworks web disponíveis têm prós e contras distintos. Vamos destacar apenas um deles aqui.

Instagram e o Washington Post estão entre os milhares de aplicativos web construídos sobre um framework web chamado Django ("O D é mudo"). Django é surpreendentemente agradável de usar. Tendo vindo de um background em Python, o fato de usar Python tornou ainda mais fácil para mim aprender. Passei pelo tutorial Escrevendo seu primeiro aplicativo Django e estava pronto para começar. Este foi realmente um momento emocionante para mim porque eu podia sentir minha mente se expandindo... "Tantas novas possibilidades. Eu posso fazer qualquer aplicativo web por aí com esta coisa!" É a coisa mais próxima de subir de nível que acho que já vou experimentar.

Django é extremamente bem documentado e tem uma comunidade divertida e engajada. Por exemplo, Django Girls é uma organização sem fins lucrativos centrada em ensinar Django para meninas ao redor do mundo enquanto desperta interesse em programação.

Nota

Um modelo mental que é frequentemente usado nesses tipos de aplicações é o padrão "Modelo-Visualizador-Controlador" (MVC). O modelo é o banco de dados organizado de alguma forma, o visualizador é a página da web renderizada com os dados nela, e o controlador é o código executado para descobrir o que mostrar quando, e o que você pode e não pode fazer. Ao separar essas unidades, o código permanece fácil de manter. Django é projetado em torno do MVC.

A magnitude pura das capacidades que o Django e seus complementos trazem é impressionante. Uma vez que você chega ao ponto de definir modelos, muitos sistemas impressionantes são ativados automaticamente. Por exemplo, digamos que você esteja fazendo um fórum de discussão onde você criou alguns modelos que se parecem com isso:

class Post(models.Model):
    title = models.CharField(max_length=300)
    poster = models.ForeignKey(User)
    posted = models.DateTimeField()
    votes = models.IntField(default=0)

class Comment(models.Model):
    title = models.CharField(max_length=300)
    poster = models.ForeignKey(User)
    post = models.ForeignKey(Post)
    posted = models.DateTimeField()
    votes = models.IntField(default=0)

O painel de administração automático permitiria que você populasse o banco de dados com informações. Ele apresenta a você widgets baseados na web que são apropriados para cada tipo de dado. O DateTimeField tem um seletor de calendário pop-up e os ForeignKeys têm menus suspensos com links para outras instâncias. Você pode criar formulários personalizados para seus usuários que automaticamente obtêm esses widgets também. Você também pode usar o framework Django REST (um complemento) que configurará todo um sistema para interagir com este aplicativo de outros lugares (outros aplicativos web, programas de desktop, telefones, etc.). Em outras palavras, todas as coisas que você normalmente precisa fazer com um aplicativo web têm soluções integradas ou prontamente disponíveis.

Eu simplesmente não posso enfatizar o quão capacitador isso é. Se você aprender, começará a querer automatizar todos os tipos de ineficiências ao redor do seu escritório, equipe de pesquisa, armazém, ou onde quer que as encontre. A documentação oficial não pode ser melhorada e, para começar, basta ir até lá e começar. Ter Python instalado da seção anterior é tudo o que você precisa para começar. Estou ansioso para ouvir sobre as coisas maravilhosas que você fará com esses sistemas.

Nota

Habilidades de administração de sistemas para executar um servidor web são úteis para configurar aplicativos Django de calibre de produção, embora você possa começar a construir aplicativos imediatamente usando o servidor de teste embutido que vem com o Django.
